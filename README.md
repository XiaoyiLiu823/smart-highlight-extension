# Smart Keyword Highlighter

一个面向长文本阅读场景的 Chrome 插件。  
通过 **关键词高亮 + 自动高频词发现 + 词频分层显示**，帮助用户在长文、论文、技术文档中快速识别核心概念，降低通读前的信息筛查成本。

---

## 项目简介

在阅读长文本时，用户通常需要先通读一遍，才能大致判断文章反复在讨论什么。  
对于英文论文、技术博客、长篇网页内容，这一过程尤其耗时。

本项目的核心思路是：

- 用户可自定义关键词列表，对重点词进行人工标记
- 插件自动统计页面中的高频词
- 根据频次阈值进行分层高亮
- 通过颜色深浅和词频排序，将长文本中的“主题骨架”直接显化出来

相比普通统一高亮，这个插件更强调：

- **频次结构可视化**
- **长文本重点提取**
- **参数可调的阅读辅助**

---

## 适用场景

- arXiv / 论文阅读
- 技术博客 / 长篇英文文章
- API 文档 / 产品文档
- 政策文件 / 报告类网页
- 需要快速判断文章重点的阅读场景

## 核心功能

### 1. 手动关键词分层高亮
用户可以输入关键词列表，对指定词进行人工标记。

高亮规则：

- 在关键词列表中，且出现次数 **小于 x**：浅绿色
- 在关键词列表中，且出现次数 **大于等于 x**：深绿色，并显示词频角标

这部分用于标记用户预先关注的概念。

---

### 2. 自动高频词发现
对于 **不在关键词列表中** 的词，插件自动统计全文频次。  
满足条件的高频词将进入黄色高亮层。

规则：

- 不在关键词列表中
- 不在停用词表中
- 不在排除词列表中
- 出现次数 **大于等于 y**

这部分用于帮助用户发现“文本自己反复在强调什么”。

---

### 3. 高频词颜色分档
自动高频词不是统一黄色，而是会根据频次继续分档。  
用户可设置一个步长，例如 `30`，则：

- `y ~ y+29`：浅黄
- `y+30 ~ y+59`：黄色
- `y+60 ~ y+89`：深黄偏橙
- `y+90 ~ y+119`：橘色
- 更高频次：更深橘色

这样可以直接在正文中区分“高频”和“更高频”的概念。

---

### 4. 词频排序面板
插件支持根据用户设定的阈值 `n`，输出出现次数 **大于等于 n** 的词，并按频次降序排序。

示例：

- qwen — 96
- user — 84
- model — 81
- assistant — 72

该功能可以帮助用户在不通读全文的前提下，快速把握文章主要讨论对象。

---

### 5. 排除词列表
用户可手动指定不希望高亮的词，例如：

- said
- people
- one
- 这个
- 但是

排除词优先级高于其他高亮规则，可用于去噪。

---

### 6. 会话内自动高亮
支持“会话内自动高亮”开关：

- 开启后，本次浏览器会话中，新打开的网页会自动套用当前参数进行高亮
- 关闭浏览器后失效

该功能适合连续阅读多篇文献或长文时使用。

---

## 功能价值

本项目的价值不只是“高亮关键词”，而是通过 **词频分层** 把长文本中的核心概念直接显化出来。

实际使用中，用户可以：

- 先设一个较低阈值，观察页面中有哪些高频词
- 再逐步提高阈值，过滤噪音词
- 让最终保留下来的高频词接近文章的“主题骨架”

例如，在论文页面中，随着阈值提高，最终只剩下：

- qwen
- user
- model
- assistant

即使不先通读摘要，也能快速判断文章主要在讨论什么。

对于英文阅读能力一般、但需要高效筛选文献的用户，这种“单词级主题提取”比直接阅读摘要更低门槛。
